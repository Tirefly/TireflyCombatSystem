## 执行顺序依赖图

```
Phase A（语义对齐）──┐
                     ├── Phase B（基础设施）─── Phase C（Attribute 迁移）─── Phase D（生命周期迁移）─── Phase E（应用/槽位/查询）
Phase A 可独立执行                                                                                                    │
                                                                                                                       v
                                                                                                             Phase F（deprecated 标记）
                                                                                                                       │
                                                                                                                       v
                                                                                                             Phase G（硬删除 + 清理）
```

**关键约束**:
- Phase A 建议作为独立 pre-PR 先合入（纯机械操作，零风险，缩小后续 diff）
- Phase B 必须先于 C/D/E（Component 需要 lazy resolve helper 和 Manager 的 Allocate API）
- **Phase C 必须先于 Phase D**（Phase D 的 `FinalizeStateRemoval` Step 5 依赖 Phase C-1 迁移产物 `UTcsAttributeComponent::RemoveModifiersBySourceHandle`，不可并行）
- Phase D 必须先于 E（生命周期方法是槽位链路的依赖）
- Phase E 内部（E-1 状态应用 / E-2 槽位链路 / E-4 查询）可在 D 合入后拆任务并行
- Phase F 必须在 C+D+E 全部完成后
- Phase G 必须在 F 之后，且是最终步骤
